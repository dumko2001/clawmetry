FAQ

Common questions

Everything about ClawMetry + NemoClaw.

Do I need to change my NemoClaw agent code? ▾
No. ClawMetry reads your agent's session logs and workspace files directly. It connects to the agent gateway — no SDK, no decorators, no changes to your agent code required.
Does ClawMetry send my agent data anywhere? ▾
No. ClawMetry runs entirely on your machine. All data stays local. The dashboard is served at localhost:8900 — nothing leaves your environment unless you explicitly enable cloud sync (opt-in, separate feature).
Which NVIDIA NeMo versions are supported? ▾
ClawMetry works with NeMo Agent Toolkit (NemoClaw) via the standard gateway interface. We test against the latest stable releases. If you hit a compatibility issue, open a GitHub issue — we respond fast.
Can I use ClawMetry with other agent frameworks too? ▾
Yes. ClawMetry started as an OpenClaw observability tool but now supports any agent stack that produces OpenTelemetry traces or standard session log files. NemoClaw, LangGraph, CrewAI, AutoGen — all work.
Is ClawMetry really free? ▾
Yes — ClawMetry OSS is MIT licensed and free forever. We also offer ClawMetry Cloud (opt-in, paid) for teams that want multi-node fleet management and centralized dashboards. The local dashboard is always free.
How is this different from NeMo's built-in logging? ▾
NeMo's built-in logging gives you trace-level call data. ClawMetry adds the operational layer on top: real-time cost tracking, budget alerts, sub-agent trees, cron health, memory file diffs, and a live browser dashboard — all without touching your agent code.
